> 작성자: Claude Sonnet 4.6

## 1. 배경 및 목적

현재 앱의 핫스팟 분석은 200m 격자 기반 집계에 의존한다.  
ML 클러스터링 라이브러리를 활용해 **분석 정확도 향상 + 비정상 패턴 자동 감지**로 업그레이드한다.  
모든 기능은 **완전 오프라인 동작**, **외부 라이브러리 최소화**를 전제로 한다.

---

## 2. 아키텍처

Hugging Face 모델 추론은 적용하지 않는다.  
순수 JS 연산만 사용하므로 인터넷 연결, GPU, 별도 빌드 과정이 불필요하다.

| 방식 | 패키지 | 특징 |
|------|--------|------|
| **순수 JS ML** | `density-clustering` | 완전 오프라인, 추가 빌드 없음, 어떤 PC에서나 동작 |
| **순수 JS 통계** | 없음 (내장 구현) | 외부 의존성 0 |

---

## 3. 적용 기능 2가지

### Feature 1 — 단속 핫스팟 클러스터링 (격자 → DBSCAN)

**문제**  
현재 격자 방식은 인위적인 경계선 때문에 실제 단속 밀집지와 격자가 어긋남.  
도로 경계에 걸리는 마커가 서로 다른 셀로 분리되어 핫스팟이 과소평가됨.

**해결**  
`density-clustering`으로 마커를 Haversine 거리 기반으로 자연스럽게 클러스터링.  
격자 없이 반경 200m, 최소 3개 포인트 기준 DBSCAN 적용.

**패키지**  
- `density-clustering` v1.3.0 — npm, 로컬, 추가 빌드 없음

**결과 비교**

```
현재: 200m 격자로 잘라서 집계 (격자 경계에 걸리면 마커가 분리됨)
개선: DBSCAN eps=200m minPts=3 → 도로 흐름에 맞는 자연스러운 군집
```

**기존 코드 영향**  
`_analyzeMarkers()` 내부 격자 집계 로직만 교체. UI 변경 없음.

---

### Feature 2 — 시계열 이상감지 (비정상 급증 배지)

**문제**  
특정 구간 단속이 갑자기 폭증해도 수동으로 데이터를 확인해야만 감지 가능.

**해결**  
구간별 단속 건수 시계열에서 통계적 이상값(Z-score) 자동 감지 후 배지 표시.

**패키지**  
- 없음 — 이동평균 + 표준편차 순수 JS 구현

**감지 방식**  
```
이동평균(4주) ± 2σ 초과 시 → "⚠️ 비정상 급증" 배지 핫스팟 목록에 표시
```

**기존 코드 영향**  
핫스팟 목록 렌더링(`_renderList()`)에 배지만 추가. DB/분석 로직 변경 없음.

## 5. 기술적 고려사항

### 시스템 요구사항

| 항목 | 요구사항 |
|------|---------|
| 인터넷 | 불필요 |
| GPU | 불필요 |

| 추가 빌드 | 없음 (`electron-rebuild` 불필요) |

### 추가 파일 설계

```
src/
└── ai/
    └── ai-analysis.js      ← DBSCAN, Z-score 로직
```

### 데이터 흐름

```
vectorSource.getFeatures()          ← 현재 마커
        ↓
[src/ai/ai-analysis.js]             ← 새 모듈 (IPC 핸들러로 노출)
        ├─ DBSCAN 클러스터링
        └─ Z-score 이상감지
        ↓
AnalysisManager._renderPanel()      ← 기존 패널에 결과 표시
```
